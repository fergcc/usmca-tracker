# USMCA / T-MEC / CUSMA Negotiation Tracker

A self-hosted intelligence monitor for the 2026–2036 USMCA annual joint-review
cycle. It polls public, machine-readable sources on a schedule, filters against
your Boolean queries, scores each item for signal, de-duplicates against a local
database, and pushes only *new* items to you.

It was built from — and corrects — the strategy brief you drafted. The core
premises checked out against primary sources (see "What the tracker assumes"
below).

---

## What you get

| File | Purpose |
|---|---|
| `usmca_tracker.py` | The tracker. Run once or on a schedule. |
| `config.yaml` | All sources, queries, keywords, notifications. Edit this, not the code. |
| `build_dashboard.py` | Builds `dashboard.html` from `data/items.jsonl` — a filterable, searchable web view of every dispatch. Run after any real run. |
| `requirements.txt` | Optional Python packages (`feedparser`, `PyYAML`). |
| `com.scientika.usmca-tracker.plist` | macOS launchd job for near-real-time polling. |
| `README.md` | This file. |

Outputs it creates as it runs:

- `briefings/usmca-brief-YYYY-MM-DD_HHMM.md` — a dated brief of each run's new items, sorted by signal.
- `data/items.jsonl` — append-only machine log of every item (feed your own tools from this).
- `data/seen.db` — SQLite dedupe memory so you're never pinged twice.
- macOS desktop banner for each batch of new items.
- `dashboard.html` — generated by `build_dashboard.py`, not by the tracker itself.

## Quick start (macOS)

```bash
# 1. Put the files in a folder, e.g.
mkdir -p ~/usmca-tracker && cd ~/usmca-tracker
# (copy all the delivered files here)

# 2. Install the two optional-but-recommended packages
python3 -m pip install -r requirements.txt

# 3. See what it would fetch WITHOUT saving or notifying
python3 usmca_tracker.py --config config.yaml --dry-run

# 4. Do a real run (writes a brief, updates the db, sends a banner)
python3 usmca_tracker.py --config config.yaml
```

## Near-real-time scheduling (the "event-driven" part)

**Honest framing:** none of these sources push events to you. "Real-time"
here means *polling frequently* — every 15 minutes by default. That's as
close to real-time as a self-hosted scraper gets without paid streaming APIs.

**macOS (launchd, recommended):**

```bash
# Edit the three REPLACE_ME paths inside the plist first, then:
cp com.scientika.usmca-tracker.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.scientika.usmca-tracker.plist
# It runs immediately, then every 15 min. To stop:
launchctl unload ~/Library/LaunchAgents/com.scientika.usmca-tracker.plist
```

Change `StartInterval` in the plist (900 = 15 min) to poll faster or calmer.

**Any OS (cron), every 15 min:**

```
*/15 * * * * cd ~/usmca-tracker && /usr/bin/python3 usmca_tracker.py --config config.yaml >> data/cron.log 2>&1
```

## The sources, and what each can and can't do

| Source | Access | Reality |
|---|---|---|
| **Federal Register** | ✅ Free JSON API, no key | The gold layer. Public-comment notices & tariff changes post here *before* they take effect. Auto-scored high. |
| **Congress.gov** | ✅ Free JSON API, requires a key ([sign up](https://api.congress.gov/sign-up/)) | Bills & resolutions mentioning USMCA/CUSMA/T-MEC — the legislative branch's side of the story (e.g. "Protecting the USMCA from Harmful Chinese Investment Act"). The API has **no full-text search**, so the tracker pages through every bill in the current Congress (~17k, ~70 API calls) and filters locally by title. That means a relevant bill whose title doesn't mention USMCA/trade explicitly can be missed. Auto-scored moderately high (primary legislative source, but not yet binding law). |
| **Google News RSS** | ✅ Free | The workhorse. Runs your Boolean queries and per-outlet `site:` searches. Returns headlines + snippets + links. |
| **USTR / Global Affairs Canada** | ✅ via Google News `site:` | Official reactions surface as headlines; links open the real page. |
| **Diario Oficial de la Federación (MX)** | ⚠️ via Google News `site:` | No clean API. The `site:dof.gob.mx` query is a pragmatic proxy; for legal certainty, still check the DOF site directly for the day's edition. |
| **CSIS / Rethink Trade** | ✅ via Google News `site:` | Think-tank analysis surfaces as headlines with links. |
| **Inside U.S. Trade** | 🔒 Subscription | Headlines/snippets surface via `site:insidetrade.com`, but **opening the full article needs your paid login**. The tracker cannot (and should not) scrape behind their paywall. |
| **X / social (Greer, Ebrard, Sidhu)** | 🔒 Paid API | Not included — X's API is now paid and scraping violates ToS. Follow their accounts manually, or add a paid provider later. |

If a source ever fails (network blip, layout change), the run logs a `[warn]`
and keeps going — one bad feed never kills the batch. This was verified.

## Tuning it

Everything lives in `config.yaml`:

- **`queries`** — your three Boolean intelligence lanes (macro, roundtables, China). Edit the `google_news` line to change what's monitored. The `boolean` line is the canonical form to paste into Feedly/Inoreader/Google Alerts if you also want those.
- **`site_feeds`** — add/remove outlets by changing the `site:` domain.
- **`scoring`** — words that raise an item's signal score (`critical` +3, `players` +2, `sectors` +1). A Federal Register item gets +3 automatically, a Congress.gov bill gets +2. Add a rival's name, a new tariff line, a company — it'll bubble to the top of the brief and (if enabled) trigger email.
- **`federal_register.terms`** — the terms queried against the FR API.
- **`congress.terms`** — the terms matched locally against bill titles. `congress.max_pages` bounds how much of the current Congress gets crawled (80 pages × 250 bills covers it with room to spare). Requires `CONGRESS_API_KEY` in your environment — get one free at [api.congress.gov/sign-up](https://api.congress.gov/sign-up/); **never put it in `config.yaml`**.
- **`notifications.email`** — off by default. To turn on: set `enabled: true`, fill SMTP details, and export your password as an env var (`export USMCA_SMTP_PASSWORD=...`). `min_score` keeps low-signal noise out of your inbox. **Never put the password in the file.**

Print your Boolean queries for use in other tools:

```bash
python3 usmca_tracker.py --print-queries
```

## What the tracker assumes (verified July 2026)

Checked against primary sources before building, so the monitor isn't
propagating a false premise:

- On **July 1, 2026** the US declined the clean 16-year renewal; this triggered the **annual joint-review process under Article 34.7.4**, running each year until the agreement would expire **July 1, 2036** unless extended. The 16-year extension remains available at any time by written confirmation of the three leaders.
- US–Mexico bilateral rounds: **1st** May 28–29 (Mexico City), **2nd** June 16–17 (Washington), **3rd** scheduled for the **week of July 20** (Mexico City), moving to detailed legal-text drafting.
- Key principals: **Jamieson Greer** (USTR) / Dep. **Jeff Goettman**; **Marcelo Ebrard** (MX Economy); **Mark Carney** (PM) / **Maninder Sidhu** (CA Trade).

One correction to the original brief: it's better described as *"annual reviews
running through a 2036 expiry horizon"* than a fixed "10-year cycle" — the
annual reviews continue only until the parties either extend or let it lapse.

## A few honest cautions

- **This is a monitoring aid, not a source of truth.** For anything you'd act or advise on, open the linked primary document (Federal Register notice, USTR release, DOF edition) and read it.
- **Google News coverage is broad but not exhaustive** and its ranking changes. Pair it with the Federal Register layer (which *is* authoritative) for legal/procedural moves.
- **Respect paywalls and ToS.** The tracker surfaces public headlines; it does not defeat subscriptions. Inside U.S. Trade's value is real — a paid seat is the legitimate way in.
- **Keep secrets out of files.** SMTP and Congress.gov API keys go in environment variables, never in `config.yaml` or the plist committed anywhere.
- **Congress.gov has no full-text search.** The tracker crawls every bill title in the current Congress rather than searching, so a relevant bill with an indirect title (no "USMCA"/"CUSMA"/"T-MEC" in it) can be missed. Cross-check congress.gov's own search UI periodically if this matters to you.
